# Kuraudo セキュリティ監査レポート

**監査日**: 2026年5月10日（v3.0 同期バックエンド対応版）
**対象**: Kuraudo v0.1.0 全ソースコード  
**監査範囲**: 暗号化実装、鍵管理、データ保護、認証、通信、情報漏洩、同期バックエンド

---

## 開発方針

Kuraudo は個人が開発・維持するオープンソースの無料アプリです。セキュリティ修正は対応しますが、定期的なアップデートは保証しません。気になる点は自分でビルドして使用することを推奨します（GPL-3.0）。

---

## 総合評価

暗号化の基本設計（Argon2id + AES-256-GCM）は堅実で、業界標準に準拠しています。ファイルフォーマットの設計も適切です。

## ✅ 修正済み

### 1. OAuthクライアントシークレットのハードコード ✅

**ファイル**: `google_drive_service.dart`  
**対応**: `--dart-define=GOOGLE_CLIENT_SECRET=xxx` でビルド時注入に変更。ソースコードにシークレットは含まれない。

### 2. OAuthトークンの平文保存 ✅

**ファイル**: `google_drive_service.dart`  
**対応**: `flutter_secure_storage` による暗号化保存に変更済み。

### 3. デバッグprint文の残存 ✅

**ファイル**: `crypto_engine.dart`, `lock_screen.dart`  
**対応**: 全 `print` 文を削除済み。

### 4. 派生鍵のメモリクリアが不完全 ✅

**ファイル**: `crypto_engine.dart` `deriveKey()`  
**対応**: `passwordBytes.fillRange(0, passwordBytes.length, 0)` を追加済み。

## 🟡 既知の制限事項（Dart/Flutter言語の制約）

### 5. マスターパスワードがメモリに平文保持

**ファイル**: `vault_service.dart`  
**内容**: `_masterPassword` が String 型で保持される  
**理由**: Dart の文字列はイミュータブルでGCに依存するため、明示的なゼロ化が言語仕様上不可能  
**現状対応**: `lock()` 時に `_masterPassword = null` で参照を切っている（これが Dart で取れる最善策）  
**補足**: 端末がroot化されていなければ実用上のリスクは限定的

### 5. Argon2idパラメータ強化 ✅

**ファイル**: `crypto_engine.dart` KdfParams  
**対応**: デスクトップ: 64MB/3回/4並列、モバイル: 32MB/3回/2並列に変更済み（OWASP推奨水準）  
**補足**: ファイルヘッダーにKDFパラメータを保存しているため、旧パラメータで作成したファイルも後方互換で読み込み可能

### 6. パスワード強度評価の強化 ✅

**ファイル**: `password_generator.dart` `evaluateStrength()`  
**対応**: 辞書攻撃耐性を追加。よく使われるパスワード（トップ50）、キーボードパターン、辞書単語、リート表記（p@ssw0rd等）を検出してスコアを減算

### 7. PIN認証の試行回数制限 ✅

**ファイル**: `lock_screen.dart` `_verifyPin()`  
**対応**: 5回失敗でPIN入力を無効化し、マスターパスワードによる解除を強制。ロック解除後にカウンターリセット

### 8. エクスポート時のセキュリティ強化 ✅

**ファイル**: `settings_screen.dart` `_export()`  
**対応**: エクスポート前にマスターパスワードの再確認を要求。クリップボード経由時は30秒後に自動クリア。ファイル保存時は「ファイルを削除」アクションを表示

## 🟢 許容済みの軽微な項目

### 9. 設定ファイルが平文保存

**ファイル**: `main.dart`  
**内容**: `kuraudo_settings.json` に PIN 有効/無効フラグ、自動ロック時間、選択中の同期バックエンド種別等が平文保存  
**判断**: 設定ファイルにパスワード・認証情報・トークンは含まれない。Vaultフォルダ内に置かれるため、Vaultが取られる時点で本質的な情報は既に漏洩していると考えられる。許容可能。
**v3.0変更**: ドキュメントフォルダではなく Vault と同じフォルダに保存するよう変更。OS既定領域には何も書き込まない方針。

### 10. クリップボードクリアのタイミング ✅

**ファイル**: `main.dart`, `autofill_service.dart`, `MainActivity.kt`  
**対応**: Android 9+で`clearPrimaryClip()`による履歴ごと完全削除。バックグラウンド移行時・アプリ終了時に即座クリア。設定画面にON/OFFトグル追加。30秒タイマーも完全クリアに変更

### 11. ファイルヘッダーのHMAC不足

**ファイル**: `kuraudo_file.dart`  
**内容**: ヘッダー改ざん検出用HMACがない  
**判断**: AES-GCM の認証タグが復号時に改ざんを検出するため、実用上の影響はない。

## 🔵 同期バックエンド（v3.0）のセキュリティ考慮

### 12. WebDAV認証情報の保管 ✅

**ファイル**: `webdav_backend.dart`
**内容**: サーバーURL、ユーザー名、パスワード、リモートパスを SecureStorage に保存
**判断**: OS のキーストア経由で暗号化保管されるため、平文ファイル化は避けられている。Android Keystore / Linux Secret Service / Windows Credential Manager を使用。

### 13. WebDAV通信の暗号化

**ファイル**: `webdav_backend.dart`
**内容**: HTTPS通信を推奨（HTTPでも技術的には接続可能）
**判断**: アップロードされるファイルは既に AES-256-GCM で暗号化済みのため、HTTP接続でもファイル内容そのものは漏洩しない。ただし HTTP では Basic 認証のユーザー名・パスワードが平文で流れるため、HTTPS 強制を推奨。
**今後の課題**: `WebDAVBackend.configure()` で HTTP URL を検出した際の警告表示を検討。

### 14. WebDAV自己署名証明書

**ファイル**: `webdav_backend.dart`
**内容**: 自己署名証明書のサーバーへの接続は現状非対応
**判断**: TOFU（Trust On First Use）方式での証明書ピン留めなど、適切な実装が必要なため、安易にバイパスを許可していない。
**今後の課題**: 必要に応じて実装。

### 15. ローカルパス同期先のアクセス権

**ファイル**: `local_path_backend.dart`
**内容**: ユーザー指定ディレクトリにファイルをコピー。ディレクトリ自動作成あり
**判断**: ファイルパーミッションは OS のデフォルトに依存（ユーザーがマウント時に管理する責務）。Vault は AES-256-GCM 暗号化済みのため、ファイルが他者に読まれてもマスターパスワードがなければ復号不可。

### 16. 同期バックエンド切り替え時の旧情報

**ファイル**: `main.dart`
**内容**: バックエンド切り替え時、旧バックエンドの認証情報は SecureStorage に残ったまま
**判断**: ユーザーが意図的に切り替えただけかもしれないので、自動削除はしない。明示的な「切断」操作で削除される設計。
**注意点**: 端末譲渡前は各バックエンドの「切断」を実行することを推奨。

### 17. WebDAV接続情報の名前空間衝突

**ファイル**: `webdav_backend.dart`
**内容**: SecureStorage キーは `webdav_*`、`localpath_*`、`gdrive_*`（既存）でプレフィックス分離済み
**判断**: 安全。複数バックエンドの認証情報が混ざることはない。

## ✅ 設計の良好な点

- **AES-256-GCM**: 認証付き暗号化で改ざん検出が組み込まれている
- **Argon2id**: 最新の鍵導出関数（bcrypt/PBKDF2より優秀）
- **ソルト/ノンス**: 暗号化ごとにランダム生成、再利用なし
- **ゼロ知識設計**: サーバーにパスワードを一切送信しない
- **OAuthトークン**: `flutter_secure_storage` で暗号化保存
- **OAuthシークレット**: ビルド時注入（ソースに含まれない）
- **クリップボード自動クリア**: 30秒で実装済み（エクスポート時も同様）
- **外部依存の最小化**: 暗号化は純Dart実装（`argon2` + `pointycastle`）のみ使用
- **PIN総当たり対策**: 5回失敗でマスターパスワード解除を強制
- **エクスポート保護**: マスターパスワード再確認 + ファイル削除オプション
- **辞書攻撃対応の強度評価**: よく使われるPW・パターン・リート表記を検出
- **OS非依存領域への情報漏れなし（v3.0）**: ドキュメントフォルダ等の OS 既定領域を使わず、設定・バックアップは Vault と同じフォルダ、認証情報・パスポインタは OS のキーストアに格納
- **複数バックエンド対応のキー分離**: SecureStorage のキーをバックエンド毎にプレフィックスで分離（混在防止）
