# Kuraudo — Google Drive OAuth設定ガイド

## これは何？

Kuraudoの「クラウド同期」機能を動かすには、Googleに「このアプリがユーザーのDriveにアクセスしていいですよ」と許可を取る仕組み（OAuth）が必要です。

**一言でいうと：** Google Cloud ConsoleでプロジェクトとAPIキーを作り、アプリに設定するだけです。

---

## 全体の流れ（5ステップ）

```
① Google Cloud Consoleでプロジェクト作成（3分）
② Google Drive APIを有効化（1分）
③ OAuth同意画面を設定（5分）
④ OAuthクライアントIDを作成（5分）
⑤ アプリに設定ファイルを配置（2分）
```

---

## ① Google Cloud Consoleでプロジェクト作成

1. ブラウザで開く: https://console.cloud.google.com/
2. Googleアカウントでログイン
3. 画面上部の **プロジェクト選択** ドロップダウン → **「新しいプロジェクト」**
4. プロジェクト名: `kuraudo`（何でもOK）
5. **「作成」** をクリック
6. 作成したプロジェクトが選択されていることを確認

---

## ② Google Drive APIを有効化

1. 左メニュー → **「APIとサービス」** → **「ライブラリ」**
2. 検索バーに **`Google Drive API`** と入力
3. 「Google Drive API」をクリック → **「有効にする」** ボタン
4. 有効化されると「管理」画面に遷移する → OK

---

## ③ OAuth同意画面を設定

これは「ユーザーがGoogleアカウントでログインするときに表示される画面」の設定です。

1. 左メニュー → **「APIとサービス」** → **「OAuth同意画面」**
2. **User Type**:
   - 自分だけで使う/テスト段階 → **「外部」** を選択
   - **「作成」** をクリック
3. **アプリ情報**:
   - アプリ名: `Kuraudo`
   - ユーザーサポートメール: 自分のメール
   - デベロッパー連絡先: 自分のメール
   - 他は空欄でOK → **「保存して次へ」**
4. **スコープ**:
   - **「スコープを追加または削除」** をクリック
   - フィルタに `drive` と入力
   - `.../auth/drive.appdata` にチェック（これが「アプリ専用の隠しフォルダ」へのアクセス権）
   - もし見つからなければ `.../auth/drive.file` にチェック
   - **「更新」** → **「保存して次へ」**
5. **テストユーザー**:
   - **「+ ADD USERS」** をクリック
   - **自分のGmailアドレスを追加**（これ重要！テスト段階では登録したユーザーしか使えない）
   - **「保存して次へ」**
6. 概要を確認 → **「ダッシュボードに戻る」**

---

## ④ OAuthクライアントIDを作成

ここがプラットフォーム別に少し違います。

### Linux（デスクトップ）の場合

1. 左メニュー → **「APIとサービス」** → **「認証情報」**
2. **「+ 認証情報を作成」** → **「OAuthクライアントID」**
3. アプリケーションの種類: **「デスクトップアプリ」**
4. 名前: `Kuraudo Desktop`（何でもOK）
5. **「作成」**
6. **クライアントIDとクライアントシークレットが表示される**
   - この2つをメモする（後で使う）
   - 「JSONをダウンロード」もしておく

### Androidの場合

1. まずアプリの**SHA-1フィンガープリント**を取得:
   ```bash
   # デバッグ用（開発中はこれでOK）
   keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
   ```
   → `SHA1: XX:XX:XX:...` の行をコピー

2. **「+ 認証情報を作成」** → **「OAuthクライアントID」**
3. アプリケーションの種類: **「Android」**
4. パッケージ名: `com.zerotoship.kuraudo`
5. SHA-1証明書フィンガープリント: ↑でコピーしたもの
6. **「作成」**

### Windowsの場合

Linuxと同じ「デスクトップアプリ」でOK。同じクライアントIDを共用できます。

## ⑤ アプリに設定を配置

### Linux/Windows（デスクトップ）

`google_sign_in` パッケージはデスクトップでは**環境変数**または**コード内の設定**でクライアントIDを渡します。

プロジェクトルートに `.env` ファイルを作成:

```bash
cp .env.example .env
```

`.env` を編集:

```
GOOGLE_CLIENT_ID_DESKTOP=あなたのクライアントID.apps.googleusercontent.com
```

> **注意**: 現在のKuraudoのコードは `google_sign_in` パッケージを使用しています。
> Linux/Windowsでの `google_sign_in` は制限がある場合があります。
> その場合は `google_drive_service.dart` でHTTP直接認証に切り替える修正が必要になる可能性があります。

### Android

1. Google Cloud Console → 「認証情報」 → 作成したAndroidクライアントID
2. **JSONをダウンロード**は不要（`google_sign_in` が自動で処理）
3. ただし `google-services.json` が必要な場合:
   - Firebase Consoleでプロジェクトを作成
   - Androidアプリを追加（パッケージ名: `com.zerotoship.kuraudo`）
   - `google-services.json` をダウンロード
   - `android/app/` に配置

## テスト手順

### 1. ビルドして起動
```bash
flutter run -d linux
```

### 2. Vaultを作成してエントリを追加
- マスターパスワードを設定
- テスト用のエントリを2-3個作成

### 3. 同期をテスト
- 三点メニュー → **「クラウド同期」**
- **「Googleアカウントでサインイン」** をクリック
- ブラウザが開いてGoogleログイン画面が出る
- ③で登録したテストユーザーのアカウントでログイン
- 「このアプリは確認されていません」と警告が出る → **「詳細」** → **「Kuraudo（安全でない）に移動」**
- 権限を許可
- アプリに戻る → サインイン成功

### 4. 同期操作を試す
- **「ローカル → クラウド」**: データがDriveにアップロードされる
- **「マージ同期（v2.0）」**: UUID単位でマージ
- Google Driveを開いて「アプリデータ」にファイルがあることを確認
  （注: appDataFolderは通常のDrive画面には表示されません。APIからのみアクセス可能）

## トラブルシューティング

### 「サインインに失敗しました」

- テストユーザーに自分のメールが追加されているか確認
- OAuth同意画面のステータスが「テスト」になっているか確認
- Google Drive APIが有効になっているか確認

### 「このアプリは確認されていません」の警告

- **正常です。** テスト段階ではGoogleの審査を受けていないため表示されます
- 「詳細」→「（アプリ名）に移動」でスキップできます
- Store公開前にGoogleの審査（OAuth verification）を申請すれば消えます

### Linuxで `google_sign_in` が動かない

`google_sign_in` はLinuxでの対応が限定的です。動かない場合:
1. `google_drive_service.dart` でOAuth2のHTTPフロー（`localhost`リダイレクト）に書き換えが必要
2. もしくはAndroid実機でのみ同期をテストする

### 「アクセスがブロックされました」

- OAuthクライアントIDのタイプが正しいか確認（デスクトップ/Android）
- パッケージ名とSHA-1が一致しているか確認（Android）

---

## 公開時に必要な追加作業

テスト段階では「テストユーザー」しか使えませんが、Store公開時には:

1. OAuth同意画面のステータスを **「本番」** に変更
2. Googleの **OAuth verification**（審査）を申請
   - プライバシーポリシーURL必須（`PRIVACY.md` をGitHub Pagesで公開）
   - アプリのスクリーンショット必須
   - 審査に1-4週間かかる
3. 審査通過後、全Googleユーザーが利用可能になる

## 概念図

```
┌──────────────────┐     OAuth認証      ┌──────────────────┐
│                  │ ──────────────────→ │                  │
│   Kuraudo アプリ  │                     │  Google Cloud    │
│   (Flutter)      │ ←────────────────── │  (OAuth Server)  │
│                  │   アクセストークン    │                  │
└──────────────────┘                     └──────────────────┘
         │                                        │
         │ Drive API (暗号化済み.kuraudoファイル)     │
         ↓                                        ↓
┌──────────────────┐                     ┌──────────────────┐
│ ローカルファイル    │ ←──── 同期 ────→   │ Google Drive     │
│ ~/Documents/     │                     │ (ユーザーの      │
│ kuraudo.kuraudo  │                     │  appDataFolder)  │
└──────────────────┘                     └──────────────────┘
```

**重要**: Googleのサーバーに送られるのは**暗号化済みのバイナリファイル**です。マスターパスワードはGoogleに送信されません。Google側からファイルの中身を読むことは不可能です。
